## Applied Lessons
1. Non-custodial game systems need explicit architectural and UX guardrails.
2. Chain/metadata signals must be bounded and smoothed before mechanic mapping.
3. Provenance and stale-state visibility are core trust features in live-data games.
4. Deterministic debug hooks are required for balancing and regression workflows.
5. Fallback profiles should preserve playability under API degradation.
